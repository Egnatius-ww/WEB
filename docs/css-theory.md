# Теория CSS по материалам проекта BuildMart

Документ привязан к тому, **что реально встречается** в `styles/*.css` этого репозитория: так проще объяснять «каждую строчку» в терминах стандарта, а не заучивать справочник целиком.

---

## 1. Как читать одну строку CSS

Синтаксис декларации:

```css
свойство: значение;
```

- **Свойство** — что меняем (например `display`, `margin`).
- **Значение** — одно или несколько через пробел/запятую (`transition: color 0.2s ease, transform 0.2s ease`).
- **`;`** — конец декларации внутри блока `{ ... }`.
- **Комментарий** — `/* ... */`, браузер игнорирует.

**Селектор** перед блоком задаёт, *к каким элементам* применяются декларации:

```css
.catalog__toolbar { ... }           /* класс */
a:hover { ... }                    /* псевдокласс */
.header__mobile-panel::before { } /* псевдоэлемент */
```

## 2. Каскад (cascade), порядок и специфичность

### Каскад

Несколько правил могут целиться в один элемент. Итог собирается **слоями**:

1. **Происхождение** (user agent → пользователь → автор страницы).
2. **Важность** — декларация с `!important` перебивает обычную (в проекте есть, напр. у `display: none !important` для скрытого бейджа корзины).
3. **Контекст** (например анимация).
4. **Специфичность** селектора.
5. **Порядок в файле** — при равной специфичности побеждает то, что **ниже**.

*Пример из проекта:* старый блок с `position: absolute` для каталога, затем ниже — переопределение на `position: relative` и `width: 100%`: нижнее правило выигрывает при равной/большей специфичности или за счёт порядка.

### Специфичность (упрощённо)

Считают «вес»: ID > класс/атрибут/псевдокласс > тег/псевдоэлемент.

- `.header__cart-badge[hidden]` — два класса + атрибут → выше, чем один класс.
- `#cartPage .cart-page__item` — ID даёт большой вес.

**Комбинаторы** (` ` потомок, `>` дочерний, `+` сосед) сами по себе вес не добавляют, но удлиняют цепочку классов.

### Наследование (inheritance)

Часть свойств **наследуется** детьми (`color`, `font-family`), часть **нет** (`margin`, `border`, `width`).  
Для наследуемых дочерний элемент берёт значение от предка, пока не задано своё.

`inherit` / `initial` / `unset` в проекте могут не встречаться — но это стандартные ключевые слова «взять у родителя / сбросить».

## 3. Бокс-модель и `box-sizing`

Каждый элемент — прямоугольник: **content → padding → border → margin**.

- **`box-sizing: border-box`** (везде в проекте) — `width`/`height` включают **padding и border**, удобно для вёрстки по макету без «раздува» ширины.
- **`box-sizing: content-box`** (дефолт браузера) — `width` только про content.

**Ответ в одну строку:** `border-box` — ширина блока равна заявленной ширине, внутрь уже «вложены» отступы и рамка.

---

## 4. Единицы измерения

| Единица | Смысл |
|--------|--------|
| `px` | Пиксели экрана, абсолютная привязка к растровой сетке. |
| `%` | Процент от **контейнга** (часто ширины родителя; для `margin`/`padding` — особые правила). |
| `rem` | От корневого `html` (удобно для масштаба типографики). |
| `em` | От **текущего** `font-size` элемента (вложенность накапливается). |
| `vw`, `vh`, `dvh` | Доля viewport; `dvh` — динамическая высота (мобильный браузер с адресной строкой). |
| `fr` | Только в **grid** — доля свободного места в сетке. |

В проекте также: **`clamp(min, preferred, max)`** — «живое» значение между полами; **`calc()`** — выражения; **`min()`/`max()`** — выбор меньшего/большего.

---

## 5. Display: поток и роли блоков

- **`block`** — на всю строку (в пределах контенера), новая строка.
- **`inline`** — в строке текста, ширина по содержимому.
- **`inline-block`** — гибрид (в строке, но можно задать размеры).
- **`flex`** — flex-контейнер; дети — flex-элементы.
- **`grid`** — grid-контейнер; дети выкладываются в ячейки.
- **`none`** — не рисуется в layout, не занимает место (отличие от `visibility: hidden`).

---

## 6. Flexbox (много в `header`, `footer`, `main`, карточках)

**Ось:** по умолчанию главная ось — горизонталь (`flex-direction: row`).

Частые свойства контейнера:

| Свойство | Зачем |
|----------|--------|
| `display: flex` | Включить flex-контекст. |
| `flex-direction` | `row` / `column` — направление главной оси. |
| `flex-wrap` | Перенос строк (`wrap` в тулбаре каталога). |
| `justify-content` | Выравнивание **вдоль главной** оси (`flex-start`, `space-between`, `center`). |
| `align-items` | Выравнивание **поперёк** главной оси в одной линии. |
| `align-content` | Если несколько линий flex — распределение **линий** между собой. |
| `gap` | Зазор между элементами (и по строке, и по колонке). |

Свойства элемента:

| Свойство | Зачем |
|----------|--------|
| `flex: 1 0 auto` | Сокращение для `flex-grow`, `flex-shrink`, `flex-basis` (в `main` для растягивания контента). |
| `flex-shrink: 0` | Не сжиматься при нехватке места (иконки, бейджи). |
| `min-width: 0` | Разрешает flex-ребёнку **сжаться меньше контента** без переполнения (важно для `text-overflow` / длинного текста). |

**Кратко:** flex — одномерная раскладка (строка или колонка), удобно для выравнивания и «прижать вправо» через `margin-left: auto`.

---

## 7. CSS Grid (каталог товары, корзина, карточка)

Контейнер:

- `display: grid`
- `grid-template-columns` — шаблон колонок (`1fr`, `repeat(3, minmax(0, 1fr))`, фиксированные `px`).
- `grid-template-rows` — строки (реже в проекте).
- `gap` — расстояние между **ячейками**.

**`minmax(0, 1fr)`** — колонка может сжиматься до 0 на уровне трека (в связке с `min-width: 0` у детей борется с переполнением).

**`repeat(n, ...)`** — повтор шаблона.

**Выравнивание в ячейке:** `justify-items`, `align-items`, или у элемента `justify-self` / `align-self`.

**Кратко:** grid — двумерная сетка; `fr` делит свободное место; `minmax` задаёт минимум и максимум трека.

---

## 8. Позиционирование и слои

| Значение | Смысл |
|----------|--------|
| `static` | Обычный поток (дефолт). |
| `relative` | Сдвиг относительно своего места в потоке; создаёт контекст для абсолютных детей. |
| `absolute` | Вырывается из потока; позиция относительно ближайшего **позиционированного** предка. |
| `fixed` | Относительно viewport (в проекте реже). |
| `sticky` | Гибрид: в потоке, пока не достигнут `top`/`bottom` при скролле — «прилипает» (хедер). |

**`z-index`** работает только в **stacking context** (создают: `position` не `static`, `opacity` < 1, `transform`, `filter`, `isolation`, и др.). Больше число — «ближе к зрителю».

---

## 9. Отступы, размеры, переполнение

- **`margin`** — снаружи border; может схлопываться по вертикали с соседним margin.
- **`padding`** — внутри border.
- **`width` / `height` / max- / min-** — ограничения размера.
- **`overflow: hidden | auto | visible`** — что делать, если контент не влезает.
- **`aspect-ratio`** — фиксировать пропорции блока (галерея карточки товара).

---

## 10. Типографика и цвет

- **`font-family`, `font-size`, `font-weight`, `line-height`, `letter-spacing`** — шрифт и ритм.
- **`color`** — цвет текста (наследуется).
- **`text-align`, `text-decoration`, `white-space`, `text-overflow`** — поведение текста; `nowrap` запрещает переносы.
- **`background`, `background-color`, `border-radius`** — фон и скругления.

**Цветовые модели в проекте:** hex `#101828`, `rgba(...)`.

---

## 11. Границы, тени, визуальные эффекты

- **`border`, `border-width`, `border-color`, `border-style`**
- **`outline`** — линия **вне** border, не влияет на размеры; часто для `:focus-visible`.
- **`box-shadow`** — тень блока.

---

## 12. Псевдоклассы (состояние элемента)

Примеры из проекта:

| Селектор | Когда |
|----------|--------|
| `:hover` | Указатель над элементом (на десктопе). |
| `:focus`, `:focus-visible` | Фокус клавиатуры/программный; `:focus-visible` — только когда браузер считает нужным показать обводку. |
| `:active` | Момент нажатия. |
| `:first-child`, `:last-child`, `:nth-child(n)` | Позиция среди соседей у одного родителя. |
| `:not([hidden])` | Элемент без атрибута `hidden`. |
| `:empty` | Нет детей (редко). |

**`:hover` на тачскрине** — может «залипать» до тапа вне; для критичного UX иногда разделяют медиа `@media (hover: hover)`.

---

## 13. Псевдоэлементы и сложные селекторы

- **`::before` / `::after`** — виртуальные дети; нужен `content: ""` для отображения; часто декоративная линия (`header__mobile-panel::before`).
- **`::placeholder`** — стиль плейсхолдера в `input`.
- **`::-webkit-details-marker`** — убирает стрелку у `<details>` в WebKit.

**Комбинированные селекторы:** `.a.b` — элемент с **обоими** классами; `.a .b` — `.b` внутри `.a`.

---

## 14. Атрибутные селекторы

Пример: `[hidden]`, `[aria-checked="true"]` — стиль по HTML-атрибуту. Удобно для доступности и состояний без лишних классов.

---

## 15. Медиазапросы (`@media`)

Условие: «если viewport подходит — применить блок».

Типичные формы в проекте:

- `min-width: 768px` — **от** ширины и выше.
- `max-width: 767px` — **до** ширины включительно.
- `min-width: 768px and max-width: 1023px` — диапазон.
- `width: 1024px` — **ровно** одна ширина (используется для точечной подгонки сетки корзины).

**Mobile-first vs desktop-first:** в одном файле может быть смесь; важно помнить **порядок** и перекрытие правил.

---

## 16. `@keyframes` и анимация

В корзине: `@keyframes cart-page-promo-hint-in` + `animation: ...` для подсказки промокода.

Свойства анимации: `animation-name`, `duration`, `timing-function`, `delay`, `iteration-count`, `direction`, `fill-mode`.

---

## 17. Переходы (`transition`)

Плавная смена **между состояниями** (например цвет при `:hover`), а не по ключевым кадрам.

Синтаксис: `transition: свойство длительность easing[, …]`.

**Что анимируется безопасно для производительности:** в основном `transform` и `opacity` (композитинг GPU). Анимация `width`/`height`/`margin` тяжелее — в проекте для аккордеона на карточке используется **`max-height`** (компромисс).

---

## 18. Трансформации и фильтры

- **`transform: translate(...)`, `scale(...)`, `rotate(...)`** — визуальный сдвиг без смены потока (до тех пор, пока не влияет на layout через другие механизмы).
- **`transform` на детях** — часто для иконок, галереи, тостов.

---

## 19. Кастомные свойства (`--*` и `var()`)

```css
:root {
  --line: #e4e7ec;
}
border-color: var(--line, #e4e7ec);
```

- **`--line`** — переменная, живёт на элементе и наследуется вниз.
- **`var(--line, #fallback)`** — если переменной нет, подставится fallback.

В `card.css` есть размеры галереи через переменные — одна точка правки.

## 20. Функции раскладки и математики

| Функция | Назначение |
|---------|------------|
| `calc(a + b)` | Сложение единиц (`100% - 32px`). |
| `min(a, b)` / `max(a, b)` | Выбрать меньшее/большее. |
| `clamp(lo, mid, hi)` | Ограничить «живое» значение снизу и сверху. |
| `minmax(min, max)` | В Grid — границы трека. |

---

## 21. Формы и нативные контролы

- Стили **`input[type="range"]`** в каталоге через **псевдоэлементы WebKit/Moz** (`::-webkit-slider-thumb`, `::-moz-range-track`, …) — это **vendor-specific** слои поверх разных движков.
- **`appearance`** (если встречается) — сброс нативного оформления.

---

## 22. Доступность и UX в CSS

- **`:focus-visible`** — обводка не «мешает» мыши, но видна с клавиатуры.
- **`aria-hidden`** — не CSS, но селектор `[aria-hidden="true"]` может сопровождать скрытие от вспомогательных технологий (осторожно: не прятать фокусируемое).
- **`prefers-reduced-motion`** — в JS карточки для аккордеона; в CSS можно сокращать анимации тем же медиа.

## 23. Именование (BEM в проекте)

Паттерн: **блок `__элемент` — `_модификатор`**.

Примеры: `.catalog__product-button`, `.header__navigation-link--categories`.

**Зачем:** предсказуемые классы, низкий риск коллизий, удобно искать в CSS/HTML.

## 24. Шпаргалка: «одна строка — что сказать устно»

| Свойство | Одна фраза |
|----------|------------|
| `display: flex` | Включаю flex-контекст, выравниваю детей по осям. |
| `justify-content` | Распределение вдоль главной оси flex/grid. |
| `align-items` | Выравнивание поперёк главной оси в одной линии. |
| `gap` | Фиксированный зазор между flex/grid-элементами. |
| `flex: 1 0 auto` | Можно расти, не сжиматься с нуля, база `auto`. |
| `min-width: 0` | Разрешаю узкий flex/grid-ребёнок без переполнения. |
| `grid-template-columns` | Задаю ширины колонок сетки. |
| `minmax(0, 1fr)` | Колонка делит место, но может сжиматься до нуля на уровне трека. |
| `position: sticky` | В потоке, но прилипаю к `top` при прокрутке. |
| `position: absolute` | Вынимаю из потока, позиционирую относительно предка с не-static позицией. |
| `z-index` | Порядок слоёв внутри stacking context. |
| `box-sizing: border-box` | Ширина/высота включают padding и border. |
| `width: 100%` | На всю ширину контейнера (с учётом box-sizing). |
| `max-width` | Не шире значения; удобно для `1280px` контейнеров. |
| `margin-inline: auto` | Горизонтальные отступы авто — центрирование блочного элемента с ограниченной шириной. |
| `overflow: auto` | Скролл, если контент не влезает. |
| `object-fit: cover` | Картинка заполняет рамку с обрезкой, сохраняя пропорции. |
| `aspect-ratio` | Фиксирую соотношение сторон блока. |
| `transition` | Плавно меняю указанные свойства между состояниями. |
| `transform` | Двигаю/масштабирую/вращаю без смены места в потоке. |
| `!important` | Принудительно выигрываю у обычного каскада (использовать редко). |
| `@media (min-width: …)` | Правила только от указанной ширины viewport. |
| `var(--x, fallback)` | Подставляю кастомное свойство или запасное значение. |
| `clamp(a, b, c)` | Беру среднее выражение, но не выхожу за min/max. |
| `calc()` | Считаю значение из смеси единиц. |

## 25. Как отвечать «по строке» на экзамене

1. Назвать **селектор** — *кого красим*.
2. Сказать **свойство** — *что меняем в модели визуализации*.
3. Объяснить **значение** — *как именно* (единицы, ось flex, трек grid).
4. При необходимости — **взаимодействие с соседними правилами**: каскад, медиа, специфичность.

Пример:  
`grid-template-columns: repeat(3, minmax(0, 1fr));`  
→ «Три равные колонки в grid; `1fr` делит свободное место; `minmax(0, 1fr)` позволяет треку сжиматься до нуля на уровне сетки, чтобы не ломать вёрстку при длинном контенте внутри ячеек».
